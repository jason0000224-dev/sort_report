合併排序演算法解說

C 語言實作

1void merge(int arr[], int left, int mid, int right) {
2    int i = left, j = mid + 1, k = 0;       // 指標初始化
3    int temp[right - left + 1];
4    while (i <= mid && j <= right) {        // 比較兩側
5        if (arr[i] <= arr[j]) {
6            temp[k++] = arr[i++];              // 左側較小，取左
7        } else {
8            temp[k++] = arr[j++];              // 右側較小，取右
9        }
10    }
11    while (i <= mid) temp[k++] = arr[i++]; // 複製剩餘左側
12    while (j <= right) temp[k++] = arr[j++];
13    for (i = left, k = 0; i <= right; i++, k++) {
14        arr[i] = temp[k];                  // 複製回 arr
15    }
16}
17void merge_sort(int arr[], int left, int right) {
18    if (left < right) {
19        int mid = left + (right - left) / 2;  // 避免溢位
20        merge_sort(arr, left, mid);            // 分割左半
21        merge_sort(arr, mid + 1, right);       // 分割右半
22        merge(arr, left, mid, right);          // 合併
23    }
24}

複雜度分析

時間複雜度
  最佳 O(n log n)
  平均 O(n log n)
  最差 O(n log n)

空間複雜度
  O(n)         ← 需要額外空間

穩定性
  穩定（Stable）

特性
  分治演算法、時間複雜度穩定
  需要額外記憶空間、快速排序
  適合大型資料集合

核心概念

分治法：將陣列「分割」成左右兩半（遞歸）→ 對兩半分別排序 → 「合併」兩個已排序的部分。
最底層將單個元素視為已排序，從底向上逐次合併。時間複雜度恆定 O(n log n)，但需額外 O(n) 記憶空間。
